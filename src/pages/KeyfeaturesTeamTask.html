Key features

Create task

To add a new task, click “Create Task”.

For more detail, check Task Form Overview (2.4.1)

Edit Inline

eTask provides a quick and easy way to edit tasks in an “Excel-like” mode.

In the Grid View, click on “Edit” button on the tool bar. When the edit mode is activated, the other buttons on the tool bar, Group by, and Filter function will be disabled, “Expanded Bug grid” and “Bug counter” will be hidden (if user has eBug).

Click on the desired fields to edit.

Some fields are not allowed to be edited:

  • ID
  • Source
  • Bucket
  • Phase
  • Estimation
  • Modified
  • Created By

Mouse cursor will change to the “Block” sign.

Changed fields will have the red triangle like this:

When there is any change in the Edit mode, “Save” button will be available. Click on “Save” at the top-right to save changes or click on “Cancel” to undo the changes.

eTask allows the “Edit” mode in one page at a time, if user switches to the next page while editing, a pop-up window will be shown:

  • Cancel: Turn off the pop-up window.
  • Don’t Save: Do not save changes and go to the next page.
  • Save: Save and go to the next page.

Bulk Edit

In Task View, Bulk edit is a function which supports editing multiple tasks at once. Bulk edit allows user to edit the following fields:

  • Assigned To
  • Status
  • Priority
  • Start Date
  • Due Date

By default, “Bulk Edit” button will be disabled. User can tick on any box of desired items to enable the “Bulk Edit” button.

Note: The selected items will be deselected when users change page or uses the function Filter/Group By.

Click on “Bulk Edit” to open edit form. Input the values and click “Save Changes” to apply to all selected items. Click “Cancel” to return to grid.

Notification Messages

There are 3 types of message:

  • Successfully
    • Pop-up a message and then 4 seconds later, it fades out.
  • 1 success and 1 failure
    • Pop-up a message, list out the ID of error items and success items, have a button “Got it” to confirm user have read the message.
  • All failed.
    • Pop-up a message, list out the ID of all error items, have a button “Got it” to confirm user have read the message.

Refresh

User can click to get the newest update of the view.

Sort

Click on the column header to sort based on that column. If user clicks on the same column header, it will sort the column in the opposite order.

Paging

Click on the page number or the arrows at the bottom right of the tab to navigate through the pages.

Show/Hide columns

Click on the “eye” icon to show a list of all the columns user can show/hide. It’s only possible to show more columns if there’s enough space in the eTask window. Any excess columns will be hidden.

Simple search

Enter text into the Search box to perform a quick search on all columns that are searchable (version 1.0 only supports the Title column).

Note: Search is only available in Task View and Board View, but not Chart View.

Filter

Click Filter to show filter options

After setting the filters as desired, click "Apply" to apply the filters to the Task list. Note that filters only persist during the active session.

Date

Select Start Date or Due Date in “Search by”, then enter the date range. To clear the date range, click "Clear Date".

Priority

Select Priority values to filter.

Status

Select Status values to filter.

Assigned To

Type user name or email to find a user. For accounts from other sources that have been mapped, O365 account will appear for filtering. It’s possible to assign to multiple users.

Source

Only the data sources that have been configured for this Channel are available for filter.

Reset to Default Filters

Click "Reset to default filter" to reset to the filter settings configured when the view was created.

Group by

Click on the … and select "Group By", then choose a desired column.

Note: If there are more than one page, each "group by" value might not show all tasks for that value in one page. Tasks are split across pages so that users can see different groups by items in a page, instead of having to navigate through many pages to find the next "group by".
