Layout & Access Data

  • New Dashboard setting

    Title: Enter the display name of your dashboard for the list here

    Chart Configuration

    Chart configuration will set the layout of the chart widget. User can toggle to show or hide chart elements (legend, series label, tooltips) and pick where they should be.

    Once everything is decided. Click Filter.

    Note:

    You can filter which information user want to take into account.

    If user want information using the default filter setting, skip this Filter section and click Save.

  • Filter for PM Dashboard

    Choose Projects:

    List out all active projects, select the check-boxes inside dropdown to pick which project user want to take into account.

    Standardize Color Mapping By:

    Choose a primary project to get color mapping of that project.

    Note:

    If user unselect the primary project, Standardize Color Mapping By will automatically pick the first project in the remain selected list in Choose Projects.

    User cannot click Save for these cases:

    • If user is currently not in any management team, Choose Projects will be blank.
    • If no project is selected in Choose Projects, Standardize Color Mapping By will be blank.

    Date:

    • Filter By: User can pick Start Date or Due Date
    • Time Frame: The amount of time that will be take into account
    Note:

    By default,

    Filter By value = Start Date

    Time Frame value = “Recent Months”

    Recent Month includes “Last Month / Current Month / Next Month”

    Priority, Status, Assigned To:

    This works similar to eTask grid view, Click Save to create the Dashboard.

  • Widgets:

    • Sticky report

      • Projects: Show the total number of projects of that PM is working on.
      • Total tasks: Show the total number of tasks of these projects have in Time Frame.
      • Members: Show the total number of members of the projects except those who don’t have any task during the Time Frame. If a member is in multiple projects, it is still counted as 1.
      • Overdue: Show the total number of tasks that have status = “Overdue” of these projects. This will include task that has due date with:
        • Status = “Completed”: Actual completed date > Due Date (later).
        • Other statuses: Current Date > Due Date
      • In-Progress: Show the total number of tasks that have status = “In-Progress” of these projects.
      • Not Started: Show the total number of tasks that have status = “Not Started” of these projects.
      • Type: None.

    • Project list

      Shows the projects with status = “Active” in briefly details. There are columns display the statistic of project:

      • Project Name: Show the name of the project. This will be Project Display name or Project name (if there is no display name)
      • Tasks: Show the total tasks that include all statuses except “Completed” of these projects.
      • Bugs: Show the total bugs that include all statuses except “Closed” of these projects.
      • Type: None.

    • Team size

      • Team Size chart shows the members of a project that have all tasks statuses in current Time Frame.
      • Total member in Team Size chart will be different compare to Members in Sticky Report. Briefly information above. Because there might be 1 member working on 2-3 projects.
      • Type: Column, Bar.

    • On-time tasks

      Show percentage of tasks that are completed on time, compare to all completed tasks.

      • Type: Column, Bar.

    • Overdue by Priority

      Show chart of overdue tasks sorted by priority in all projects in current Time Frame. Priority will be sorted from top to bottom: High, Normal, Low.

      • Type: Stack.

    • Status

      Show chart of tasks with all statuses sorted by Status in all project in current Time Frame.

      • Type: Donut, Pie, Column, Bar.

    • Priority

      Show chart of tasks by Priority in all Projects in current Time Frame.

      • Type: Donut, Pie, Column, Bar.

    • Project Workload by Status

      • Shows chart of all tasks that included all statuses but sorted by Projects. Provide a quick glance for user.
      • Vertical axis: Projects.
      • Horizontal axis: Number of tasks.
      • Within the chart: sorted number of tasks followed by current Status.
      • Overdue: overdue tasks from the incomplete tasks.
      • Type: Stack.

    • Monthly workload

      • Shows the chart of tasks by Projects in Monthly. Each bar represents a project.

      • Status filter:

        • All
        • Not Started
        • In-Progress
        • Hold
        • Completed
        • Overdue
      • If Global filter enable or disable a status > In workload filter will have the same available statues as Global Filter.
      • If Global Filter disable a status being chosen in workload filter, by default, workload filter will choose option “All”.
      • Type: Column, Bar, Line.

    • Team Member via stack

      • Breakdown all tasks for each member base on status (with overdue)
      • User can click on element of the chart to show detail information grouped by projects
      • Type: Stack.

    • Team Member via Portrait

      • Breakdown all tasks for each member filtered by status
      • User can click on element of the chart to show detail information grouped by projects
      • Type: Portrait.

    • Due Soon by Priority

      Due soon task sort by priority for each project. Priority will be sorted from top to bottom: High, Normal, Low.

      • Type: Stack.

    • High priority by status

      High priority task sort by Status for each project. Status will be sorted from top to bottom: Completed, In Progress, On Hold, Not Started.

      • Type: Stack.

    • Created vs. Completed

      Show the amount of created task along with the amount of completed task by month.

      X-axis is Projects, Y-axis is number of task.

      • Type: Column, Bar, Line.

    • Rich text

      • Support basic word editor, indexing, and images.

    • iFrame

      • Insert an URL, iFrame will load the web content within widget.

  • Customize function

    User can click … of a widget to Customize it.

    Custom:
    • Title: Name of the widget
    • Types: Column, bar, pie, stack, line,…
    • Operator: Count

    • Allow scrolling:

      • Yes: All the widget will have the same height and have a scrollbar
      • No: The widget will expand to its optimized size
    Change:

    Let user choose another widget

    Delete:

    Remove the widget from the section

  • Drag & Drop Widget

    User can move a widget from 1 column to another or between sections.
