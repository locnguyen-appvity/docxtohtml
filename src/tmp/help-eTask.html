eTask Introduction

Updated by 2019/05/21

eTask is a cloud based application is designed to solve the following business problems:

  • Synchronizing data from various data sources in Jira, Visual Studio Team Service (VSTS), and Office 365 Planner in MS Teams.
  • Consolidate tasks from other sources and display tasks in one view.
  • Users can access eTask from any device such as a Smartphone, Tablet, and Personal Computers that has a web browser and internet connection.
  • Automation workflow and message extension.
  • Generating reports in the form of charts.
  • Offering multiple views (Grid view, Chart view, Board view, Calendar view, and Gantt view) for multiple purposes.

eTask User Operations Guide

Default Home Screen

eTask is a tab application within MS Teams environment. After eTask is installed and deployed, the admin can add eTask as a tab in a channel. Here’s the default screen layout:

Left Navigation

Click the hamburger at the top left to open Left Navigation.

Default View

There are 5 default views in eTask:

  • Team Tasks (Task View): Tasks are displayed in a table list format.
  • Teams Board (Board View): Tasks are displayed as cards in a multi column board, which supports drag and drop for quick action.
  • Teams Progress (Chart view): Tasks are represented in charts.
  • Team Calendar: Tasks are represented in Calendar.
  • Team Gantt: Tasks are represented in Gantt.
Public Views

Views that user creates and shares with others are listed under Public Views.

Private Views

Views that user creates for personal use are listed under Private Views.

Create view

Apart from the Default views, users can create their own views and share with the rest of the team using the following procedure:

  • Open Left Navigation.

  • In the Settings page, enter a view name, and set the rest of the view’s settings. Note that Display Columns supports drag and drop for easy ordering.

  • Configure filter setting as needed.
  • Click “Create” to save the view.

Task view
Show/Hide columns

Click on the “eye” icon to show a list of all the columns you can show/hide. It’s only possible to show more columns if there’s enough space in the eTask window. If not, the columns will be disabled.

Paging

The default view shows 50 items. Click on the page number or the arrows at the bottom right of the tab to navigate through the pages.

Click on the column header to sort based on that column. Each further click will sort the column in the opposite order.

Board view
Overall

Board View is designed to show a quick overview of tasks so that the user can make quick actions without the need to go into the details of each task. Tasks are displayed as cards in a multi column board, which supports drag and drop.

By default, Board View is grouped by Priority.

Task card
  1. Task title: click to open task detail view
  2. Assigned to (Note: only Planner tasks can have multiple assignees on Board View)
  3. Due Date:
    • Overdue tasks
    • Tasks due Today or within the next 5 days
    • Tasks due after 5 days from Today
    • Blank: No Due date
  4. Status
    • Not Started
    • In Progress
    • Hold
    • Completed
  5. Priority
    • Low
    • Normal
    • High
  6. Task Action Menu
    • Edit
    • Clone
    • Get link
    • Delete
  7. Has Attachment
  8. Source
Board view actions

You can perform the following actions in the Board View:

  1. Group by

    • Priority (default)
    • Status
    • Due Date:
      • Late: Overdue tasks.
      • Today: Tasks due today.
      • Coming Soon: tasks due in the next 3 days.
      • Future: Tasks due after next 7 days.
      • No date: No Due date.
    • Assigned To: Choose users to group by. Tasks of each user are shown as one bucket. It’s possible to add more users/buckets to the view.

  2. Add a task

    Click the + button on top of a column to add a task with that group by value.

  3. Drag and Drop

    Drag a task from one bucket to another to change its value (depending on current group by setting). For example, if currently grouped by Priority, dragging a task from the “Low” priority bucket to “High” will in effect change the Priority value of the task from Low to High.

  4. Assign task

    Click the assign icon to assign task.

  5. Load more

    By default, each bucket only shows 10 tasks. Click "Load more" to load more tasks in that bucket.

  6. Move bucket

    Click the bucket menu and select to move left/right.

  7. Show Completed tasks

    By default, Board View does not show completed tasks (except when grouped by Status). Click "Show Completed" at the bottom of the bucket to show them.

Report view
Overall

You can have a summary of your tasks using charts eTask. By default, the Team Progress view doesn’t show any data because a Report View must be configured and created first.

Create report view

Here are the steps to create a Report View:

  1. From the left navigation, click (+) next to Private Views and select Create Report View.
  2. Configure the settings
    • Title
      • Name this view: Name of the view.
    • Chart Information
      • Title: Name of the chart.
      • X-Axis: Select a column for X-Axis.
      • Types: Select chart type. The following charts are available in the report view:
        • Line
        • Column
        • Bar
        • Pie
        • Donut
      • Operators: Operation on the value series. Currently, only Count is supported.
      • Aggregates (if X-Axis is a date column): Choose to aggregate tasks by Month or by Week (refer to 2.5.7 for more details).
    • Chart Configuration
      • Allow chart title: Show/hide chart title and select its position (Top, Bottom) if shown.
      • Allow chart legend: Show/hide chart legend and select its position (Top, Bottom, Left, Right) if shown.
      • Allow chart Series label: Show/hide chart series label and select its position. Available positions depend on the type of chart.
      • Allow chart tooltips: Show/hide chart tooltips.
  3. Configure Filter as needed (refer to section Filters).
  4. After all settings have been configured, click "Create" to create the view.
  5. If you want to display this view in the default Team Progress view, set it as default.

Date integration

If a date is selected (Start Date or Due Date) for the X-Axis column, the chart will aggregate tasks by Month or by Week.

  1. By Month

    By default, the chart displays tasks in current year. If you want to filter by date, you are only allowed to filter within 12 months (for example: from Mar 01, 2019 to Mar 01, 2020).

  2. By Week

    By default, the chart displays tasks in current month.

Color mapping

If the Color mappings in Settings have never been configured and saved, the chart will generate random colors. Refer to 5.3.3 for details to configure color mapping.

Show tasks in gantt view. Only tasks that have a Start Date or Due Date will appear.

Customize task

Add task

To add a new task, click “Create Task”.

The detail task form will be displayed, and the following fields are presented (fields with asterisk * are required):

  • Title: Task name up to 250 characters.
  • Source: Only show data sources that have been configured for this Channel. Values include:
    • eTask
    • VSTS
    • JIRA
    • Planner
  • Phase: Phase values depend on the Source selected
    • eTask: No Phase, so this field is disabled.
    • VSTS: Select Iteration from configured VSTS project.
    • JIRA: Select Sprint from configured JIRA project.
    • Planner: Select Plan.
  • Bucket: Bucket values also depend on the Source selected
    • eTask: No Bucket.
    • VSTS: Select Area from configured VSTS project.
    • JIRA: No Bucket.
    • Planner: Select Bucket.
  • Assigned To: Type user name or email to find users to assign this task. Only users from the selected Source are available for select.
  • Start Date: Select Start Date & Time (Today by default).
  • Due Date: Select Due Date & Time.
  • Description: Enter description of the task (plain text if Source is Planner).
  • Priority: Select Priority. Values include: Low, Normal, High.
  • Status: Select Status. Values include: Not Started, In Progress, Completed.
  • % Completed: Enter value for % Completed.
  • Attachment: Click Add Attachment to attach a file.

Click "Save" to save the task. Note: The save button is only enabled when all required fields have values.

Add eTask to a Channel

  • Select the Channel and click the + icon to add a new tab.

  • Look for “eTask” in the search box and select eTask.

eTask Settings

Overall
  1. Select the setting in the left panel.
  2. Configuration page for each setting.
  3. Show Instructions.
  4. Click "Save" to save the configuration for the current setting.
Configuration

Configuration shows the settings for supported data sources.

  1. Office 365

    • Select to enable MS Outlook or MS Planner tasks as needed.

  2. Visual Studio

    • Input Host Name, Token, and click "Load Projects" to load all Visual Studio projects for that token.

    • Select each Project and click "Add" to add that project to eTask.

    • After the project is added, it’s possible to change the Display Name. Click the X icon to remove that project from eTask.

  3. Jira

    • Input Host Name, User name, and Password, then click "Load Projects" to load all projects of that user in Jira.

    • Select each Project and click "Add" to add that project to eTask.

    • After the project is added, it’s possible to change the Display Name. Click the X icon to remove that project from eTask.

How to grant eTask Permission and Create Access Account for MS Teams

eTask requires permissions to access your company resources in Teams, which specify role-based access using the client application's credentials/identity, and are presented to the resource at run-time as "roles" claims in the client's access token.

eTask requires your organization admin to consent to the app usage.

Steps to consent eTask, where it has the effect of granting consent to the application for all users in your organization:

  • Navigate to the App Registrations blade as a global administrator, then select the app.
  • Select "Required Permissions", and finally hit the "Grant Permissions" button at the top of the blade.
  • https://docs.microsoft.com/en-us/skype-sdk/trusted-application-api/docs/tenantadminconsent
  • https://docs.microsoft.com/en-us/azure/active-directory/develop/active-directory-devhowto-multi-tenant-overview#understanding-user-and-admin-consent

How to create token for Visual Studio Online

To integrate into Visual Studio Online, eTask requires Personal Access Tokens (PATs) to authenticate and get/update tasks. Personal access tokens essentially are alternate passwords that you create in a secure way by using your normal authentication. PATs can have expiration dates, limited scopes, and specific Visual Studio Online organizations.

Steps to create Personal Access Token for eTask:

  • Sign in to Visual Studio Online with the account that you want to use as access account.

  • Select "Security" under your profile.

  • Click "Add" to add a new personal access token.

  • Enter description and select expiration time, organization.

  • For Authorized Scopes, eTask only needs the following permissions:

    • Identity (read)
    • Project and team (read, write and manage)
    • Security (manage)
    • User profile (read)
    • Work items (full)

  • Click Create Token to create the token
  • Make sure to copy and save the token for use later, because Visual Studio Team Services doesn’t store it.
